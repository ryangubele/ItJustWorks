# It Just Works™ の使い方

## 何をするMODか、そしてなぜ

Skyrim は*シーン*の上で動いています——会話やカットシーンのような、本来は自分で終わるはずの、脚本された場面のことです。ところが、ときどき終わらないものがあり、行き詰まったシーンは、その後に続くシーンを静かに堰き止めてしまいます。エラーひとつ出して警告することもなく、クエストを、ときにはセーブ丸ごとを、こっそり壊してしまうのです。このMOD「It Just Works」は、あなたが今いるシーンを見張り、ひとつに長く留まりすぎたら知らせ、今どのシーンにいるのかをメニューで見せ、行き詰まったシーンを停止できるようにします。狙いはそれだけです——引っかかったスイッチを、セーブを失う前に捕まえること。

このMODの機能はすべて、ひとつのページから操作します。**MOD設定メニュー > It Just Works** です。各項目の説明は以下のとおりです。

インストールしたばかりなら、手短に言うと、初期設定はそのままにして遊び続け、もしひとつのシーンに長く留まりすぎたら監視機能が肩を叩いて知らせてくれるのに任せてください。以下はすべて、もっと詳しく見たいときのためのものです。

## メニューを日本語で表示する

このMODはいくつかの言語のメニュー翻訳を同梱しています——インストーラーで選んでください。Skyrim はゲームの**言語設定**に合った翻訳を読み込みます。そのため、ゲームを英語で動かしていてメニューを日本語にしたい場合、ゲームは英語ファイルを読み続け、翻訳を入れていてもメニューは英語のままになります。対処法は二つ：インストーラーの最初のステップでその言語にチェックを入れ、次のステップでそれを既定のメニュー言語として選ぶ（翻訳を英語ファイルに上書きし、あとでリネームして戻せるよう英語版の .bak を残してくれます）か、手動で `Interface\Translations\` フォルダ内のあなたの言語のファイル `fth_ItJustWorks_JAPANESE.txt` を `fth_ItJustWorks_ENGLISH.txt` にリネームし、英語ファイルを置き換えてください。

## 現在のシーン

ページ上部は、今いるシーンのライブ表示です。どのシーンにもいなければ "None" と表示されます。ページを開くと新たに読み取るので、内容が古くなることはありません。

- **シーン** - 今いるシーン。名前が利用できるときは名前（その Editor ID）で、できないときは生の ID 番号で表示されます（下のランプを参照）。
- **Form ID** - シーンの生の ID 番号。コンソールやバグ報告に必要な場合のために、常に表示されます。
- **所属クエスト** - そのシーンが属するクエスト。たいていはこちらのほうが有用で、*何が*あなたを引き止めているのかを教えてくれます。
- **シーン内の時間** - このシーンにいるおおよその時間。MODはタイマーで確認するため、答えはチェック1回分の精度で分かります。だから `~` が付いています。

## 「Editor ID 読み込み済み」ランプ

スイッチではなく状態表示ランプです。クリックしても、真実の状態に戻るだけで何も起きません。

- **点灯** - 良好。powerofthree's Tweaks が Editor ID を読み込んでいるので、シーンやクエストが名前で表示されます。
- **消灯** - 名前が無効です。代わりにすべてが ID 番号で表示されます。どちらでもMODの動作はまったく同じで、ただ読みにくくなるだけです。

名前を有効にするには、`po3_Tweaks.ini`（powerofthree's Tweaks のインストール先にあります）を開いて `Load EditorIDs = true` を設定し、Skyrim を再起動してください。ランプが点灯し、名前が表示されます。

MODは、名前が無効だと初めて気づいたときにも、一度だけ自動で知らせます。このランプはその通知の常設版です。誰かが「なぜシーンが全部番号なのか」と尋ねたとき、ヘルプスレッドで指し示せるものです。

## 操作

- **シーンを停止** - 解決策です。本当に行き詰まっている場合、これで今いるシーンを終了します。あえて2段階になっています。**シーンを停止**を一度押して準備し（メニューを閉じたときに停止する旨の行が表示されます）、もう一度押すと取り消します。停止そのものはメニューを閉じた瞬間に起こります。ゲームがそれを実行できるだけ動いているのは、そのときだけだからです。つまり、準備して、メニューを閉じれば、完了です。

  これに頼るのは、シーンが行き詰まっていると思う場合だけにしてください。正常に動いているシーンを止めると不具合が起きることがあり、行き詰まったシーンを止めると、ゲームが追いつく際に遅延していたイベントが短く一気に発生することがあります。これは想定内で、新たな不具合ではありません。

- **更新** - ページを閉じて開き直さずに、現在のシーンを今すぐ読み直します。

## 最近のシーン

通り抜けてきた直近10件のシーンを、新しい順に、それぞれのおおよその継続時間とともに表示します。「あれ、今のは何だった？」というとき、特にシーンが速すぎて捉えられなかったときに便利です。

## 監視

あなたが見張らなくて済むように、代わりに見張る部分です。

- **警告するまでの時間** - ひとつのシーンに何分いたらMODが警告するか。既定は3です。0にすると一切警告しません。
- **確認間隔** - 監視が確認する頻度（秒）。既定は30です。0にすると監視を完全に無効化します。これは、かなり後で気づくケースを想定しているので、速くある必要はありません。10〜240秒もあれば十分で、ゲームへの負荷も軽くなります。

監視が作動すると、画面の隅に短い2行が表示されます。シーンにいる時間と、それが他のシーンを妨げていること、続いてMOD名です。これを見るのにメニューを開いておく必要はありません。

## 動作を見届ける（診断ページ）

- **監視** - バックグラウンド確認が今動いているかどうかを一語で表します：**稼働中**、**起動中**（リロード直後の一瞬は正常です）、**オフ**（確認間隔を0にした場合）、**休止中**（アンインストールページでオフにした場合）。ログを開かずにMODが生きていることを確かめる方法です。
- **前回の自己修復** - MODはときどき、たいていはリロード直後に、自分の状態をそっと再同期します - たとえば、リロードをまたいで行き詰まっていたシーンをきちんと捕まえられるよう、シーンタイマーを再同期します。ここに記録があるのは正常で健全な整備（自分で直したとMODが伝えているだけ）であって、不具合ではありません。
- **診断ログ** - トラブルシューティングやバグ報告のために、MODが Papyrus ログにどれだけ書き込むか：
  - **オフ** - 何も書きません。既定値で、通常のプレイではこのままにしてください。
  - **イベント** - シーンの変化、警告、そしてMODが自己修正するたびに記録します。バグを報告するときはこれに設定してください。
  - **毎回** - 確認のたびに1行追加します（ループの鼓動、タイマーの上昇）。タイミングの問題を追うときだけ設定し、済んだら戻してください。

ログがディスクに書き出されるのは、ゲーム側で Papyrus ログ出力が有効になっている場合だけです。`Documents\My Games\Skyrim Special Edition\` にある `Skyrim.ini`（または `SkyrimCustom.ini`）に `[Papyrus]` ブロックを追加してください：

```
[Papyrus]
bEnableLogging=1
bEnableTrace=1
```

Skyrim を再起動してください。ファイルは `Documents\My Games\Skyrim Special Edition\Logs\Script\Papyrus.0.log` に生成されます。その中から `fth_IJW` を検索してください（`findstr fth_IJW Papyrus.0.log`、または `grep`）。Mod Organizer 2 を使っている場合、これは仮想のゲームフォルダではなく、実際の Documents フォルダです。

## 設定

- **現在のシーン名を表示** - ここにキーを割り当てると、押すだけでメニューを一切開かずに、今いるシーンの名前が表示されます。「今どれにいる？」を知る最速の方法です。
- **ホットキーを解除** - そのキーの割り当てを外します。ここには ESC で解除する機能がありません（このメニューでは ESC はポーズで、ゲームが競合を警告します）。だから、一度割り当てたキーを外すにはこのボタンを使います。

## 情報

バージョンです。どのビルドを遊んでいるか一目で分かります。助けを求めるときや、最新かどうか確認するときに便利です。

## オフにする、または削除する

MODを止めるのにアンインストールは必要ありません。**アンインストール**ページには **有効** スイッチがひとつだけあります。これをオフにすると、何も掃除せず、セーブにも触れずに、MODが休止します - 監視は確認をやめ、ホットキーは登録解除されます。好きなときにオンに戻せば、中断したところからそっくりそのまま再開します。これが、プレイ途中でMODを穏やかに棚上げする方法であり、そもそもこのMODが問題の原因だったのかを手軽に確かめる方法でもあります。

本当に完全に消し去りたい場合は、次の順序で削除してください。

1. アンインストールページで **オフにする**。
2. **セーブしてから**、デスクトップに終了する。
3. MOD管理ツール（Vortex、MO2、または手動）で **MODを削除する**。

必要なのは本当にこれだけです。このMODが行うことで、抜けるときにセーブが壊れることはありません - ゲームオブジェクトを人質に取らず、何も妨げず、他の何もこれに依存していません。あとに残るのは、*どの*スクリプト系MODでも残るものと同じです。スクリプトがあった場所に残る、小さく不活性なスタブです。Skyrim はそれを無視します。それすら消したいなら、MODを削除したあとで、セーブクリーナーを使ってこのスタブを一掃できます。

### セーブクリーナーについて（ReSaver）

長く続けているセーブでは、ときどきクリーナーを走らせて、入れ替えたり外したりした*他の*MODが残したスクリプトのゴミを片付けることになります - よく使われるのが **ReSaver**（FallrimTools の一部）です。その作業中、It Just Works はインストールしたままで構いません。このMODはクリーンを生き延びるように作られています - エイリアスを使わず、ワールドステートを持たず、自己修復します。通常のパスなら手を触れませんし、確認タイマーやホットキーを消してしまう荒っぽいパスでも、次にメニューを開いたときに再武装します。*この*MODへのリスクは、設計上、スクリプト系MODとして望みうるかぎり低く抑えられています。

残る注意点は、私たちではなくツールについてのものです。大切なセーブに向ける前に ReSaver が何をするかを把握しておくこと、そして手当たり次第の一掃ではなく、実際に外したものだけを狙うことです。
