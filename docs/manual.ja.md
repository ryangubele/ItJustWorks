# It Just Works™ の使い方

## 何をするか

Skyrim は会話、カットシーン、その他のスクリプト付きの場面に *シーン* を使います。ときどきシーンが終わらないことがあります。それは後から来るシーンを静かに塞ぎ - クエストが進まないのに、エラーもクラッシュもない、という状態になり得ます。この MOD は、いま入っているシーンを監視し、長く居すぎたら警告し、それが何かを見せ、詰まっているなら止められるようにします。

**短い版:** 既定値のまま遊び続けてください。アラートが出たら **Mod Configuration Menu > It Just Works** を開きます。

必要なのは **[SKSE64](https://skse.silverlock.org/)**、**[MCM Helper](https://www.nexusmods.com/skyrimspecialedition/mods/53000)**、**[powerofthree's Papyrus Extender](https://www.nexusmods.com/skyrimspecialedition/mods/22854)**、**[powerofthree's Tweaks](https://www.nexusmods.com/skyrimspecialedition/mods/51073)**（名前を ID 番号ではなく出したい場合は `Load EditorIDs = true`）です。導入メモは [MOD ページ](https://www.nexusmods.com/skyrimspecialedition/mods/185927) にあります。

メニューは 5 ページです：**シーン**、**監視**、**設定**、**診断**、**アンインストール**。

---

## メニューを別の言語で見る

MOD はメニュー翻訳を同梱します - インストーラで選んでください。Skyrim はゲームの **言語設定** に合うファイルを読みます。英語のゲーム + 別言語を入れていても、上書きしない限り英語のメニューファイルを読み続けます。

**インストーラ:** ステップ 1 で言語にチェックし、ステップ 2 で既定のメニュー言語に設定します（英語ファイルの上に書き込み、英語の `.bak` を残します）。

**手作業:** `Interface\Translations\fth_ItJustWorks_JAPANESE.txt` を `fth_ItJustWorks_ENGLISH.txt` に改名します（英語ファイルを置き換え）。

## シーン

### いま何にいるか

現在のシーンのライブ表示、または **None**。新しい読み取りにはメニューを開きます。

- **シーン内の時間** - このシーンにどれくらいいるかのおおよそ。ゲームをリロードするとリセットされます。詰まっているかどうかの合図です。
- **シーン** - 名前が使えるときは名前、そうでなければ ID 番号。
- **Form ID** - 生の ID。常に表示。コンソールやバグレポートに便利。
- **所属クエスト** - そのシーンが属するクエスト。

### シーンを停止

シーンが詰まっていると思うなら、これで終わらせます。

1. **シーンを停止** を一度押す - アームされたと確認の一行が出ます。
2. もう一度押すとキャンセル、または **メニューを閉じて** 停止。

詰まっていると思うシーンだけ止めてください。正常なものを止めると壊れることがあります。詰まったものを止めると、ゲームが追いつくあいだに遅延イベントが短く噴き出すことが（まれに）あります。

**更新** はメニューを閉じずに現在のシーンを読み直します。バニラの Skyrim では、メニュー中はゲームが通常一時停止するので、**更新** はあまり役に立たないでしょう。メニュー中もゲームを動かし続ける [Souls](https://www.nexusmods.com/skyrimspecialedition/mods/27859) のようなアンポーズ MOD を使っている場合は、メニューを開き直さずに読み直せます。

### 最近のシーン

直近 10 シーン。新しいものが先。おおよその長さ付き。上と同じ種類の概算時間です。

## 監視

あなたが見なくても見てくれます。

- **警告するまでの時間** - 1 つのシーンに何分いたらアラートするか。既定 **3**。**0** = 警告しない。
- **確認間隔** - 確認と確認のあいだの秒数。既定 **30**。**0** = 監視をオフ。

アラートは角に出る 2 行です。例：

> scene blocking others ~3m  
> See? It Just Works!

そのシーンを出る、またはシーンが変わるまで、シーンごとに一度です。トーストを見逃した？ メニューを開いてください - 表示はいま何にいてどれくらいかを示し続けます。MOD はシーンを自動では止めません。それが **シーンを停止** です。

## 設定

- **有効** - 既定でオン。オフにすると、アンインストールせずに MOD を棚上げできます。
- **現在のシーン名を表示** - キーを割り当て、押すとメニューを開かずに現在のシーン名が見られます。
- **ホットキーを解除** - 割り当てを外します。
- **診断ログ** - Papyrus ログへどれだけ書くか。通常プレイは **オフ** のまま。バグ報告時は **イベント**。タイミング問題を追うときだけ **毎回**、その後オフに戻す。パフォーマンスに影響することがあり、とくに **毎回** で顕著です。

  ログが動くのは、ゲームが Papyrus ログを書いているときだけです。`Documents\My Games\Skyrim Special Edition\` の `Skyrim.ini` または `SkyrimCustom.ini` を編集：

  ```
  [Papyrus]
  bEnableLogging=1
  bEnableTrace=1
  ```

  再起動。ログファイル：`Documents\My Games\Skyrim Special Edition\Logs\Script\Papyrus.0.log`。`fth_IJW` を検索。

## 診断

- **Editor ID 読み込み済み** - インジケーターです。点灯時は **シーン** と所属クエストに名前、消灯時は ID 番号。**Form ID** はどちらでも生の `0x…` のままです。

  名前をオンにする：`po3_Tweaks.ini` で `Load EditorIDs = true` にし、Skyrim を再起動。名前がオフだと初めて気づいたときにも、MOD は一度これを言います。Mod マネージャはデプロイや更新でそのファイルを上書きすることがあるので、`Data` に置いただけのコピーではなく、Tweaks MOD *内*のコピー（または勝つ小さな override MOD）を編集してください。**MO2：** 左ペインの MOD フォルダ、または Overwrite／優先度の高い MOD。**Vortex：** Tweaks の staging フォルダ、または override MOD。更新後は再確認。

- **監視** - 背景チェックが動いているか：
  - **稼働中** - 問題なし
  - **起動中** - リロード直後は普通
  - **遅延** - まだ動いているが、確認がいつもより遅い（忙しいゲーム）
  - **オフ（確認を無効化）** - **確認間隔** を 0 にした
  - **休止中（オフ）** - **設定** ページで **有効** がオフ

- **前回の自己修復** - MOD はときどき自分の帳簿を直します（多くはリロード後）。ここに一行あるのは普通です。

## アンインストール

**完全に外す：**

1. **設定** ページで **有効** をオフにする。
2. セーブしてデスクトップへ終了。
3. マネージャで MOD を外す（または手作業）。

プレイ途中で外しても安全です。Skyrim はセーブに小さな不活性のスクリプト stub を残すことがあります。他のスクリプト付き MOD と同じで、ゲームは無視します。任意：セーブ cleaner（例：FallrimTools の **[ReSaver](https://www.nexusmods.com/skyrimspecialedition/mods/5031)**）で外したあとに stub を掃除できます - cleaner は慎重に、本当に外したものにだけ。*他の* MOD のゴミを掃除しているあいだ、この MOD は入れたままにして構いません。
