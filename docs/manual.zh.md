# 使用 It Just Works™

## 它做什么

Skyrim 用 *场景* 处理对话、过场和其他带脚本的片段。有时场景永远不会结束。那会悄无声息地挡住后面的场景——任务卡住、没有报错、也不崩溃。本模组监视你当前所处的场景，若在某一场景中停留过久会提醒你，让你看清是哪一个，并在卡住时允许你停止它。

**简而言之：** 保持默认设置，继续游戏。若出现提醒，打开 **模组配置菜单 > It Just Works**。

需要 **[SKSE64](https://skse.silverlock.org/)**、**[MCM Helper](https://www.nexusmods.com/skyrimspecialedition/mods/53000)**、**[powerofthree's Papyrus Extender](https://www.nexusmods.com/skyrimspecialedition/mods/22854)** 以及 **[powerofthree's Tweaks](https://www.nexusmods.com/skyrimspecialedition/mods/51073)**（若希望显示名称而非 ID 数字，请设 `Load EditorIDs = true`）。安装说明见 [模组页面](https://www.nexusmods.com/skyrimspecialedition/mods/185927)。

菜单共五页：**场景**、**监视器**、**设置**、**诊断**、**卸载**。

## 让菜单显示其他语言

模组附带菜单译文——在安装器中勾选。Skyrim 会加载与游戏 **语言设置** 匹配的文件。英文游戏 + 已安装其他语言时，仍会读取英文菜单文件，除非你覆盖它。

**安装器：** 在第 1 步勾选该语言，再在第 2 步设为默认菜单语言（写入覆盖英文文件；保留英文 `.bak`）。

**手动：** 将 `Interface\Translations\fth_ItJustWorks_CHINESE.txt` 重命名为 `fth_ItJustWorks_ENGLISH.txt`（替换英文文件）。

## 场景

### 你当前所处

当前场景的实时读数，或不在任何场景时为 **None**。打开菜单可获得新读数。

- **场景中的时间** - 你在该场景中大约停留了多久；重新加载游戏会重置它。这是卡住与否的信号。
- **场景** - 名称可用时显示名称；否则为 ID 数字。
- **Form ID** - 原始 ID，始终显示。便于控制台或错误报告。
- **所属任务** - 该场景归属的任务。

### 停止场景

若你认为场景已卡住，这会结束它。

1. 按一次 **停止场景**——一行文字确认已预备。
2. 再按一次取消，或 **关闭菜单** 以停止。

只停止你认为卡住的场景。停止正常场景可能造成破坏。停止卡住的场景可能（偶尔）在游戏“追赶”时短暂喷出一批延迟事件。

**刷新** 在不关闭菜单的情况下重新读取当前场景。在原版 Skyrim 中，打开菜单时游戏通常处于暂停状态，因此 **刷新** 多半用不上。若你在使用像 [Souls](https://www.nexusmods.com/skyrimspecialedition/mods/27859) 这类取消暂停的模组，则可在不重新打开菜单的情况下刷新菜单。

### 最近的场景

最近十个场景，最新在前，附带大致时长。与上文同类的约数时间。

## 监视器

替你盯着，省得你自己盯。

- **多久后提醒** - 在单个场景中多少分钟后提醒。默认 **3**。**0** = 永不提醒。
- **检查间隔** - 检查之间的秒数。默认 **30**。**0** = 关闭监视器。

提醒是角落里的两行，例如：

> scene blocking others ~3m  
> See? It Just Works!

每个场景一次，直到你离开该场景或场景变更。错过了 toast？打开菜单——读数仍会显示你在哪里、多久了。模组不会替你停止场景；那是 **停止场景**。

## 设置

- **启用** - 默认开启。关闭它即可搁置模组而无需卸载。
- **轻松语气** - 默认开启。通知会保持轻松的语气；关闭后改为朴素文字。只改变文字，不影响模组的运作。
- **显示当前场景名称** - 绑定按键；按下即可在不打开菜单时查看当前场景名称。
- **清除热键** - 移除绑定。
- **诊断日志** - 写入 Papyrus 日志的量。正常游玩保持 **关闭**。报 bug 时用 **事件**；只有追查时序问题时用 **每次检查**，然后将其改回关闭。可能影响性能，尤其是在 **每次检查** 时。

  仅当游戏在写 Papyrus 日志时记录才会生效。在 `Documents\My Games\Skyrim Special Edition\` 中编辑 `Skyrim.ini` 或 `SkyrimCustom.ini`：

  ```
  [Papyrus]
  bEnableLogging=1
  bEnableTrace=1
  ```

  重启。日志文件：`Documents\My Games\Skyrim Special Edition\Logs\Script\Papyrus.0.log`。搜索 `fth_IJW`。

## 诊断

- **已加载 Editor ID** - 一个指示灯。点亮时 **场景** 与所属任务显示名称；熄灭时显示 ID 数字。无论哪种情况，**Form ID** 始终是原始的 `0x…`。

  开启名称：在 `po3_Tweaks.ini` 中设 `Load EditorIDs = true`，重启 Skyrim。模组在首次发现名称关闭时也会提示一次。模组管理器在部署或更新时可能覆盖该文件，因此请编辑 Tweaks 模组*内部*的副本（或一个会胜出的小型覆盖模组），而不只是 `Data` 里的零散文件。**MO2：** 左侧面板中的模组文件夹，或 Overwrite／更高优先级模组。**Vortex：** Tweaks 的 staging 文件夹，或覆盖模组；更新后请再检查。

- **监视器** - 后台检查是否在跑：
  - **运行中** - 正常
  - **唤醒中** - 重载后片刻属正常
  - **延迟** - 仍在工作，但检查比平时慢（游戏繁忙）
  - **关闭（已停用检查）** - 你把 **检查间隔** 设为 0
  - **休眠（已关闭）** - **设置** 页上的 **启用** 已关

- **最近一次自我修复** - 模组有时会修正自身账本（常在重载后）。此处有一行属于正常。

## 卸载

**彻底移除：**

1. 在 **设置** 页，关掉 **启用**。
2. 存档，退出到桌面。
3. 在管理器中移除模组（或手动）。

可在中途安全移除。与其他带脚本模组一样，Skyrim 可能在存档中留下一小段无害的脚本 stub；游戏会忽略它。可选：存档清理工具（例如 FallrimTools 中的 **[ReSaver](https://www.nexusmods.com/skyrimspecialedition/mods/5031)**）可在移除后清理 stub——请谨慎使用清理工具，只针对你打算移除的内容。清理*其他*模组残留时，可继续安装本模组。
