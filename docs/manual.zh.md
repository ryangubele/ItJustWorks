# 使用 It Just Works™

这个MOD的所有功能都从同一个页面操作：**MOD配置菜单 > It Just Works**。以下是各部分的说明。

如果你刚安装，简而言之：保持默认设置不动，继续游戏，万一你在某个场景中停留太久，就让监视器来提醒你。下面的内容都是留给你想深入了解时看的。

## 当前场景

页面顶部是你当前所处场景的实时读数，如果你不在任何场景中则显示 "None"。打开该页面时会重新读取，因此内容不会过时。

- **场景** - 你当前所处的场景，在名称可用时以名称（其 Editor ID）显示，否则以原始 ID 数字显示（见下方的指示灯）。
- **Form ID** - 场景的原始 ID 数字，始终显示，以备你在控制台或错误报告中需要。
- **所属任务** - 该场景所属的任务。通常这是更有用的名称：它告诉你*是什么*把你困住了。
- **场景中的时间** - 你处于该场景的大致时长。因为MOD是按计时器检查的，所以答案精确到一次检查，故以 `~` 表示。

## “已加载 Editor ID”指示灯

这是一个状态指示灯，而非开关——点击它只会把它拨回真实状态，不会有其他作用。

- **点亮** - 良好。powerofthree's Tweaks 正在加载 Editor ID，因此场景和任务以名称显示。
- **熄灭** - 名称已关闭；一切都改为以 ID 数字显示。无论哪种情况，MOD的运作完全相同——只是更难阅读而已。

要开启名称：打开 `po3_Tweaks.ini`（在你的 powerofthree's Tweaks 安装目录中），设置 `Load EditorIDs = true`，然后重启 Skyrim。指示灯亮起，名称随之出现。

MOD 在首次发现名称被关闭时，也会自动提示一次。这个指示灯是那条提示的常驻版本——当有人问为什么自己的场景全是数字时，你可以在求助帖里指给他看。

## 操作

- **停止场景** - 修复手段。如果你确实卡住了，这会结束你所处的场景。它特意分为两步：按一次**停止场景**来准备（会出现一行文字，确认它将在你关闭菜单时停止），再按一次则取消。真正的停止发生在你关闭菜单的那一刻，因为只有那时游戏才运行到足以让它生效。所以：准备好，关闭菜单，完成。

  只有在你认为场景已卡住时才使用它。停止一个正常运行的场景可能会造成破坏，而停止一个卡住的场景可能会在游戏“追赶进度”时触发短暂的一批被延迟的事件——这是预期行为，并非新的错误。

- **刷新** - 立即重新读取当前场景，无需关闭再重新打开页面。

## 最近的场景

你经过的最近十个场景，最新的在前，各自附有大致的持续时间。适用于“等等，我刚才那个是什么”的情况，尤其是当某个场景一闪而过、来不及看清时。

## 监视器

替你盯着、好让你不必盯着的部分。

- **多久后提醒** - 在同一场景中停留多少分钟后MOD向你发出提醒。默认为 3。设为 0 则从不提醒。
- **检查间隔** - 监视器查看的频率（秒）。默认为 30。设为 0 则完全关闭监视器。这是为数小时后才发觉的情况设计的，所以不需要快：10 到 240 秒之间绰绰有余，而且对你的游戏更轻。
- **记录到 Papyrus** - 将每次场景切换写入 Papyrus 日志。除非你在排查问题或填写错误报告，否则请保持关闭。

监视器触发时，会在屏幕角落显示两行短讯——你在该场景中停留了多久、以及它正阻塞其他场景，然后是MOD的名称。你不必打开菜单也能看到。

## 设置

- **显示当前场景名称** - 在此绑定一个按键，按下它便会弹出你当前所处场景的名称，完全无需打开菜单。这是“我现在到底在哪个场景”的最快方式。
- **清除热键** - 解除该按键的绑定。这里没有按 ESC 清除的功能（在此菜单中 ESC 是暂停，游戏会就此冲突向你发出警告），所以一旦你设置了绑定，这个按钮就是把它重新解除的方式。

## 关于

版本号，让你一眼看出自己在玩哪个构建版本——在求助或查看是否为最新时很方便。

## 关闭它，或移除它

你不必卸载就能让MOD停下。**卸载**页面上只有一个**启用**开关：关掉它，MOD 便进入休眠——监视器停止检查，热键解除注册——而不会清理任何东西，也不会动你的存档。任何时候想开就再打开它，它都会从上次停下的地方精确地继续。这是在游戏途中把它搁置的温和方式，也是一个简单的办法，用来判断它是否曾是困扰你的那个东西。

如果你确实想彻底摆脱它，请按以下顺序移除：

1. 在卸载页面**关闭它**。
2. **先存档，再退出**到桌面。
3. 在你的MOD管理器中**移除该MOD**（Vortex、MO2，或手动）。

真的就这些。这个MOD所做的一切都不会在退出时损坏存档——它不扣押任何游戏对象，不阻塞任何东西，也没有别的东西依赖它。它留下的，正是*每一个*带脚本的MOD都会留下的东西：存档中一个曾经存放其脚本的位置上、小而无害的残留桩。Skyrim 会忽略它。如果你连它也想清掉，可以在MOD移除之后，用存档清理工具把这个残留桩清扫出去。

### 关于存档清理工具（ReSaver）

存档玩久了，你总会时不时地运行一次清理工具 - **ReSaver**（FallrimTools 的一部分）是常用的那一个 - 来清除你替换或移除的*其他*MOD留下的脚本垃圾。你完全可以在做这件事时让 It Just Works 保持安装。它天生就为熬过一次清理而设计：无别名、无世界状态、自我修复。一次正常的清理不会碰到它，即便是一次会清掉它的检查计时器或热键的激进清理，它也会在你下次打开菜单时重新就位。按其设计，*这个*MOD所承受的风险，低到了带脚本的MOD所能达到的极限。

剩下的那些告诫是关于工具本身的，与我们无关：在你把 ReSaver 指向一个你珍视的存档之前，先弄清楚它会做什么；并且只针对你确实移除的东西，而不是来一次盲目的清扫。
