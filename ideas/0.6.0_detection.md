# 0.6.0 - Detection: better accuracy, less noise

**Grounded in:** *Do one thing well* (stuck scenes, better judgment, not a detector framework); *Respect the player* (fewer false alarms, still catch save-killers); *Make it observable* (which path fired, and why a jump was deferred); *Cost nothing you'd notice* (same poll cadence, two cheap time reads); *Never break a save* (advisory only - still a shovel, not auto-Stop).

**Sequencing:** Hardest idea to get right and to verify. **0.3.0 observability is shipped** - structured log, Events vs Every check, formid join key - so a two-clock judgment is auditable on a real load order. **0.3.1 SEQ** is independent (start reliability). **0.4.0 Levity** is pure presentation. **0.5.0 localized notifications** puts functional toast copy on the translation/bake rails so multi-clock alert lines (game vs real, later phrasing) do not re-open English-only corners. Detection is targeted at 0.6.0 not because it's low priority - it's the point - but because shipping it blind would violate the prime directive, and i18n rails first keep the big change smaller. Build it when you can *watch* it decide.

## The question

How do we decide a scene has been active "too long" and is probably stuck?

**Today:** one signal - wallclock real-time, default 3 min (`fSceneFirstSeen` from `GetCurrentRealTime`, alert when elapsed > threshold). Accessible, not best.

**Not the answer:** a matrix of independent detectors, per-signal enable fashion, or "a million detectors" as more systems get stuck later. Game-time and real-time are the **only two real surfaces** for *duration-of-stuck-state*. Controls / camera later re-use judgment patterns; they don't each need a new clock zoo.

## Two clocks - different quantities

| | **Game-time** (`GetCurrentGameTime`) | **Wallclock** (`GetCurrentRealTime`) |
|--|--------------------------------------|--------------------------------------|
| Measures | In-world calendar (days, float) | Real seconds this session |
| Menu pause | Frozen | Accrues (incl. AFK in menu) |
| Sleep / wait | **Jumps** | Barely moves |
| Across reload | Persists (S-0-immune) | Resets; re-baseline already ships |
| Phrasing | "~N in-game hours/days" - strong "this is wrong" | "~Nm" - familiar |

| Situation | Game-time | Wallclock |
|-----------|-----------|-----------|
| AFK in menu (benign) | correct (frozen) | false alarm if tight |
| Slept mid-scene, scene **broke** | **catches** (jump) | slow / miss |
| Slept mid-scene, scene **fine** | false alarm (jump) | correct |
| Stuck, world running (common) | catches | catches |
| Stuck, world paused / menu-locked | misses (frozen) | **catches** |
| Long *legit* scene, world running | rarely (needs game-hours) | false alarm if tight |

Game-time is the more **semantic** measure of "how long has this scene been the state of the world." Wallclock's unique job is **paused-world stuck** and as a **gate** on sleep jumps (below).

## The direction: primary + fallback (not a matrix)

**Not:** equal OR of two first-class modes, MCM "wall only / game only / both," or a product surface that teaches a detector matrix.

**Yes:**

1. **Primary - game-time.** Alert when game-time in scene exceeds `T_game` (tunable; 0 = off that path if we keep symmetry with today's warn slider).
2. **Fallback - wallclock.** Alert when wallclock in scene exceeds `T_real`, aimed at cases game-time cannot see (menu-locked / paused world). Defaults **looser** than today's 3 min (starting guess ~15–20 min) so it stops owning the false-alarm story.
3. **Sleep / wait grace (edge on the primary).** If game-time jumps over `T_game` but **wallclock since scene enter is still small**, defer the game-time alert (likely sleep/wait, not "stuck for days of world time while you sat there"). Short grace only - sleeping *during* a scene often *is* the break; we reduce one-shot false alarms without silencing game-time forever. Log when deferred (`sig=game deferred=sleep-grace` or equivalent) so tuning is greppable.
4. **One advisory alert** (still not auto-Stop). Message should name **which path** fired (game vs real) so the player and the log agree.
5. **Err toward annoying on misses.** False positive = dismissable notify. Miss = save-killer. Grace and looser wallclock buy fewer false alarms *without* building a framework.

### Policy sketch (implement against this, not against "OR union of equals")

```text
elapsed_game  = gameNow - fSceneFirstSeenGame   # stamp on scene enter
elapsed_real  = realNow - fSceneFirstSeen       # existing, with S-0 re-baseline

if elapsed_game > T_game and T_game > 0:
    if elapsed_real < T_sleep_grace:
        # jump without much wall time - defer game path this check
        log deferred sleep-grace
    else:
        alert(sig=game)

if not yet alerted this scene (or re-alert rules) and elapsed_real > T_real and T_real > 0:
    alert(sig=real)   # fallback / paused world
```

Exact composition (defer vs require both, re-alert interaction) is empirical - the **roles** are not: game primary, real fallback + grace ruler.

### Knobs (player brain budget)

| Knob | Role |
|------|------|
| Warn / game-time threshold | Primary sensitivity (0 = off primary) |
| Wallclock / backstop threshold | Fallback (0 = off fallback); default much looser than 3 min |
| Optional: sleep-grace | If exposed at all - advanced; else a fixed short constant until data demands a slider |
| Poll interval | Unchanged job |

**No** MCM matrix of detector enable flags. Two (maybe three) numbers, not a mode diagram.

## Same-pass noise fixes

### Alerts missed under a menu (confirmed)

Poll can fire while a menu is open; `Debug.Notification` is easy to miss. Inventory mid-scene (Calixto tour, etc.) accrues wallclock *and* hides the one-shot - part of why 3 min *feels* vicious.

**Fix:** periodic **re-alert** while still over threshold (wallclock cadence is fine) until scene clears or player acknowledges. Today's `bAlerted` is one-shot per scene - that has to change carefully so we don't spam.

### Long legit scenes

Season Unending-class scenes trip any *useful* tight wallclock bar. Never know author intent from clocks alone.

**Long-scene list:** known-long scenes get a **higher threshold**, not a free pass. Still watched. Data source + maintenance + later mod registration - not a third clock, just per-scene overrides on the same policy.

## Observability (already built for this)

Events log should make support possible without guessing:

- `sig=game|real` on alert fire  
- deferred sleep-grace as an Events (or Every-check) line, not silent  
- same `scene=0x…` join key  
- precise `el=` / game-time fields (reserved names in 0.3.0 idea still apply: `gt=`, `tgt=`, etc. when you fill them)

If you can't grep which path fired, you can't tune 0.6.0.

## Stance

Some false positives are unavoidable. Prefer a dismissable wrong ping over a quiet broken save. Today's 3 min wallclock-only is too aggressive mainly because of **menu-pause + one-shot miss**, not because "stuck" is wrong as a product idea. Primary game-time + loose real fallback + short sleep grace is the smallest design that attacks that without a detector zoo.

**Out of scope for 0.6.0:** control lock detectors, camera detectors, quest repair (rejected - recommend USSEP), auto-Stop, player aliases.

## Open questions

- Default `T_game` / `T_real` / `T_sleep_grace` - empirical on real LOs. Starting guesses: game-time on the order of hours-of-world (or days if sleep-sensitive); real backstop ~15–20 min; grace a few real minutes max.
- Re-alert cadence and ack (dismiss vs any menu close vs hotkey).
- Long-scene list source and whether third-party registration ships in 0.6.0 or later.
- Whether sleep-grace is a fixed constant or a hidden/advanced setting.
- Confirm `GetCurrentGameTime` under sleep / wait / pause / timescale on target runtime.
- Stamp game-time on scene enter; interaction with wallclock S-0 re-baseline (keep for real half only).

## For context: S-0 (fixed in 0.2.0)

Wallclock is session-relative and resets each launch; `fSceneFirstSeen` persists - reload while stuck made elapsed negative until re-baseline. Game-time half needs no S-0 re-baseline. Wallclock half keeps the existing heal.
