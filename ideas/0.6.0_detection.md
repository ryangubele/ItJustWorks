# 0.6.0 - Detection: better accuracy, less noise

**Grounded in:** *Do one thing well* (stuck **scenes**, better judgment, not a detector framework); *Respect the player* (fewer false alarms, still catch save-killers); *Make it observable* (which path fired, and why a jump was deferred); *Cost nothing you'd notice* (same poll cadence, two cheap time reads); *Never break a save* (advisory only - still a shovel, not auto-Stop).

**Sequencing:** The hardest change, so it lands last of the pre-1.0 work. It sits on two rails that come first: the structured observability log (so a two-clock judgment is greppable) and the localized-toast bake (so multi-clock alert copy is localized like every other line). Detection is the product point of the mod; landing it on finished rails keeps the big change smaller.

**Observability:** Detection logs every alert decision, not just successful fires - which clock fired (`sig=`), when a jump is deferred (sleep-grace), and when a check declines (`alert hold` / `why=`). The Events level is the bug-report level, so declines must be greppable there. Full dialect below.

**Release role:** **0.6.0 is the 1.0.0 release candidate.** Defaults and edge behavior are allowed to bake in public: community feedback and several patches, then a solid 1.0.0. Empiricism is not a hold-back on shipping - it is how 1.0 gets finished. Wrong defaults get patched; silent save-killers do not.

## Product boundary (what 1.0 is - and is not)

**In scope (this mod, through 1.0):** stuck **scenes** - detect duration, name the scene, hand the player a shovel (Stop). Two clocks, one advisory surface, observability.

**Out of scope for 0.6.0 / 1.0 (not "forgotten features"):**

| Topic | Why not here |
|-------|----------------|
| Control lock detectors | Annoyance / quality-of-life; not the save-shield job |
| Camera yank / stuck camera | Same |
| Quest repair | Rejected; recommend USSEP |
| Auto-Stop | Never flip the world for the player |
| Player aliases | Persistence / coupling risk |
| Long-scene allowlists / mod registration | Nice later; not required to ship two-clock judgment |
| Detector matrix / per-signal fashion MCM | Explicitly rejected |

README and page copy have always pointed at a family of stuck switches (controls, camera, scenes). **Scenes are the ship.** Controls, camera, and other annoyances fit a **sibling mod** (or later series) that can be opinionated about comfort without this mod's prime-directive bar. Explaining that honestly at 1.0 is better than bolting half-detectors into a save-safety tool.

## The question

How do we decide a scene has been active "too long" and is probably stuck?

**Today:** one signal - wallclock real-time, default 3 min (`fSceneFirstSeen` from `GetCurrentRealTime`). Accessible, noisy under menus, weak on sleep/wait jumps.

**Not the answer:** a matrix of independent detectors. Game-time and real-time are the **only two real surfaces** for *duration of stuck state*.

---

## Two clocks

| | **Game-time** (`Utility.GetCurrentGameTime`) | **Wallclock** (`Utility.GetCurrentRealTime`) |
|--|-----------------------------------------------|----------------------------------------------|
| Measures | In-world calendar (**days** as float) | Real seconds this session |
| Menu pause | Frozen | Accrues (incl. AFK in menu) |
| Sleep / wait | **Jumps** | Barely moves |
| Across reload | Persists (no S-0) | Resets; existing re-baseline stays |
| Player phrasing | in-game hours / days | `~Nm` / `~Ns` |

| Situation | Game-time | Wallclock |
|-----------|-----------|-----------|
| AFK in menu (benign) | correct (frozen) | false alarm if tight |
| Slept mid-scene, scene **broke** | **catches** (jump) | slow / miss |
| Slept mid-scene, scene **fine** | false alarm (jump) | correct |
| Stuck, world running (common) | catches | catches |
| Stuck, world paused / menu-locked | misses | **catches** |
| Long *legit* scene, world running | rarely (needs game-hours) | false alarm if tight |

**Roles:**

1. **Primary - game-time** - semantic "how long has this been the state of the world."
2. **Fallback - wallclock** - paused-world / menu-locked stuck; also the **ruler** for sleep-grace.
3. **Sleep/wait grace** - edge on primary only: game-time over threshold but almost no wall time since enter → defer game alert this check, log it, do not silence forever.

## v1 policy (implement this)

```text
# Stamps on scene enter (and on wallclock S-0 re-baseline for real only):
fSceneFirstSeen      = GetCurrentRealTime()    # existing
fSceneFirstSeenGame  = GetCurrentGameTime()    # new; days

# Every check that runs the policy (poll / MCM refresh path that evaluates alerts):
# compute both clocks up front. Wallclock read is heal-capable (reload rebaseline if
# now < stamp). v1 EXPANDS that heal onto post-alert ticks too.
# Same two cheap reads the design already budgets; hold reuses these locals (no second
# heal-capable read). CHANGELOG must name the heal-schedule expand (see Observability).
elapsed_real = now_real - fSceneFirstSeen      # heal if now < stamp (may record correction)
elapsed_game = now_game - fSceneFirstSeenGame  # days; never re-baseline on wall reset
  # if elapsed_game < 0 (should not happen): re-stamp game and log heal

T_game_days   = iWarnGameHours / 24.0          # 0 = primary off
T_real_sec    = iWarnRealMinutes * 60          # 0 = fallback off
T_grace_sec   = 180                            # fixed 3 real minutes; not a slider
T_realert_sec = iRealertMinutes * 60           # only used when bRealert; 0 with re-alert on = treat as off

# Default path: one toast per scene (today's bAlerted). Opt-in re-alert admits
# "I miss corner notices" - we do not assume that for everyone.
if bRealert and iRealertMinutes >= 1:
    due = (fLastAlertReal < 0) or (now_real - fLastAlertReal >= T_realert_sec)
else:
    due = not bAlerted                         # one-shot; bAlerted set on first fire

playing = currentScene && IsPlaying()
fired = false
deferred_grace = false

if currentScene && playing && bEnabled && due:

    if T_game_days > 0 and elapsed_game > T_game_days:
        if elapsed_real < T_grace_sec:
            log Events: deferred sleep-grace scene=… gt=… el=… grace=…
            deferred_grace = true              # latched or every-defer: pick one, document it
        else:
            alert(sig=game)
            fired = true

    if not fired and T_real_sec > 0 and elapsed_real > T_real_sec:
        alert(sig=real)
        fired = true

# Decline / hold - OUTSIDE the fire gate (not-playing and !due are unreachable inside it).
# "Would care": enabled, in a scene, at least one thr on, past that thr on its clock,
# no toast this check, not a sleep-grace defer this check. One Events line per episode.
past_game = (T_game_days > 0 and elapsed_game > T_game_days)
past_real = (T_real_sec > 0 and elapsed_real > T_real_sec)
would_care = bEnabled && currentScene && (past_game or past_real)

if would_care && not fired && not deferred_grace && not bAlertHoldLogged:
    if not playing:
        why = not-playing
    else if not due:
        why = already          # or not-due when re-alert on and cadence waiting; one label is fine
    else:
        why = …                # should be rare (grace already excluded); do not invent spam
    log Events: alert hold scene=… el=… gt=… playing=… why=…
    bAlertHoldLogged = true

# alert(sig):
#   toast path-specific prefix + neutral duration label (localized-toast rails)
#   optional Levity punchline (English) after status line
#   bAlerted = true
#   fLastAlertReal = now_real   # for re-alert cadence when enabled
#   bAlertHoldLogged = true     # fire explains episode; INVARIANT - do not clear on fire
#   log Events: alert fire sig=game|real scene=… el=… gt=… thr=…

# scene adopt (enter/leave transition that resets bAlerted):
#   bAlertHoldLogged = false
```

**Same-tick rule:** prefer **game** over **real** if both would fire. Never two status toasts in one check.

**Elapsed expand:** up-front heal-capable wallclock elapsed every policy evaluation, including after a warning already fired for this scene. Same heal as today (reload real-time rewind → re-stamp, record correction). Not a new heal class. **CHANGELOG one line** when shipping - not "diagnostics only." Upside: after a reload mid-stuck scene, the wallclock clock is honest on the next polls even if last night's toast already spent the one-shot; with re-alert off you still need the menu or a new scene for another toast, but heal/readout/timing are not stuck on a pre-reload stamp until MCM-open.

**Re-alert (opt-in, default off):**

- **Default:** one advisory toast per scene - same as today. Scene change resets.
- **Toggle off = respect the player.** Missing a corner notice is possible (inventory, busy fight); assuming every player is that addled is rude. The menu always shows scene + elapsed; they can open it any time. Manual says so.
- **Toggle on:** player admits they want repeats. **Slider** sets real minutes between toasts while still over threshold (starting range 1–30, default **5** when they turn it on - or ship slider default 5 and ignore it until toggle is on).
- No separate "ack to silence" UI. Off = silence after first; on = cadence until scene clears.
- Clears only on scene change, not on menu close.

**Long-scene list:** **not in 0.6.0 / 1.0 RC.** Looser real default + game primary is the mitigation. A known-long list (higher thresholds, not a free pass) is a post-1.0 or patch candidate if Season Unending-class noise returns.

## v1 defaults and MCM

| Knob | Setting (working ids) | Default | Range (starting) | Notes |
|------|------------------------|---------|------------------|--------|
| Game-time warn | `iWarnGameHours:Watchdog` | **2** | 0–48, step 1, `{0}h` | 0 = off primary. Stored as **whole in-game hours**; script divides by 24 for `GetCurrentGameTime` days. |
| Real-time warn | `iWarnRealMinutes:Watchdog` | **15** | 0–60, step 1, `{0}m` | 0 = off fallback. **Replaces** today's single `iWarnMinutes` (default was 3). |
| Poll | `iPollSeconds:Watchdog` | 30 | unchanged | Unchanged job |
| Repeat alerts | `bRealert:Watchdog` | **off** | toggle | Opt-in. Manual: one toast is the default; menu is always there. |
| Repeat every | `iRealertMinutes:Watchdog` | **5** | 1–30, step 1, `{0}m` | Used only when toggle is on. Grey/disabled in MCM when toggle off if easy; else ignored in script. |
| Sleep grace | (constant) | **3 min** real | — | Not exposed until data demands a slider |

**Migration:**

- Package default for real warn moves **3 → 15**. Players who never touched the slider get the new default (Config). Players with a value in `MCM/Settings` keep their override (including 3).
- New key `iWarnGameHours` defaults to 2; missing user key → Config (same upgrade story as toast language).
- `bRealert` default off; `iRealertMinutes` default 5 in Config so the first time someone enables the toggle they already have a sane cadence.
- Prefer renaming the real key to `iWarnRealMinutes` in Config/MCM and reading it in script; if a one-release alias of old `iWarnMinutes` is trivial, use it - do not invent a save-file migrator.
- Help text must say which clock each slider is, that 0 disables that path only, and that repeat alerts are optional.

**Watchdog page brain budget:** game warn + real warn + poll + re-alert toggle + re-alert minutes. No mode enum, no detector matrix, no grace slider. Manual states the default (one toast) and points at the menu as the always-available readout.

**Docs:** player manual Watchdog section - one toast by default; turn on repeat if you want nagging; open the menu any time to see scene + elapsed without waiting for a corner notice.

## Toast and readout copy

Functional lines go through localized toast keys (bake + language index). English **source** for the tables:

| Path | Status line (prefix + neutral duration) |
|------|----------------------------------------|
| `sig=real` | `scene blocking others ` + `ElapsedLabel(elapsed_real)` → e.g. `scene blocking others ~15m` |
| `sig=game` | `scene blocking others ` + game duration label → e.g. `scene blocking others ~2h game` |

Game duration label (language-neutral suffix ok; localize the prefix via tables):

- under 2 in-game hours: whole minutes of game time if useful, else `~Nh game` / `~N game days` for large spans  
- keep it short (Skyrim shrinks long notifications); two-line pattern stays: status then optional Levity punchline  

Exact key names live with the localized-toast tables (`$fth_IJW_Toast_…`); detection needs **at least two** alert prefixes (game vs real), not one.

**MCM elapsed readout:** show both when in a scene, e.g. `~12m · ~1h game` (real first, familiar). Reload note on real half only stays in help.

**Log fields (Events / Every check):**

- `sig=game|real` on fire  
- `deferred=sleep-grace` (or `sig=game deferred=sleep-grace`) when grace holds  
- `scene=0x…`, `el=` real seconds, `gt=` game days or hours (pick one unit and stick to it in the log - prefer `gt_h=` hours float or int for grepping)  
- `thr=` for the path that fired  
- Poll tick (Every check): **`playing=`**; extend with game elapsed / dual thr only if the line stays greppable and short  
- Events **hold / decline** lines: **required** this release - see Observability below; same `scene=` join key, no third dialect

## Persistence and S-0

| Stamp | On enter | Across reload |
|-------|----------|----------------|
| `fSceneFirstSeen` (real) | `GetCurrentRealTime()` | Existing: if `now < stamp`, re-baseline, log heal |
| `fSceneFirstSeenGame` | `GetCurrentGameTime()` | **No** wallclock-style re-baseline; game calendar does not reset with the session |

`fLastAlertReal` (or equivalent): reset on scene change; session-relative like other real stamps - if negative/weird after reload mid-scene, treat as due (allow re-alert). Do not invent complex cross-reload toast suppression.

## `GetCurrentGameTime` (accepted for v1)

- Returns **days** (float) on the target runtime.  
- Frozen while menus pause the world; jumps on wait/sleep; scales with timescale.  
- v1 does not special-case timescale extremes; if community finds broken edge cases, patch.  
- Confirm once in playtest before tag - an implementation check, not a design question.

## Observability

Events must answer *why this check did or did not toast*, not only when one fired. The manual points bug reporters at **Events**, and the poll heartbeat (Every check) does not reach that level - so the decline and defer lines have to be Events-level.

The structured log dialect already exists. Detection extends it: Events must be able to grep `sig=`, sleep-grace **defer**, and **hold** declines; the Every-check heartbeat gains `playing=`.

### Events: fire, defer, and hold

The product decision is alert-or-not. Logging only successful fire is asymmetric once gates multiply (two clocks, grace, re-alert `due`, `IsPlaying()`). Ship three greppable shapes, one dialect:

| Line | When |
|------|------|
| `alert fire … sig=game\|real` | Toast went out |
| `alert … deferred=sleep-grace` (or equivalent) | Primary would fire but grace holds this check - latched or every-defer; pick one, document it; never silent forever |
| `alert hold … why=…` | Past a threshold (or otherwise "would care") but no toast and not a sleep-grace defer - **one line per episode** |

**`why=` vocabulary (v1, extend only when a new gate appears):**

| `why` | Meaning |
|-------|---------|
| `already` | One-shot spent (`bAlerted` / not `due` under re-alert off) |
| `not-playing` | Scene ref present, `IsPlaying()` false |
| `not-due` | Re-alert on but cadence not elapsed (optional if `already` would mislead) |
| (prefer `deferred=sleep-grace` over `why=grace`) | Keep grace on the defer line so fire/hold/defer stay distinct |

Warn-off paths, dormant, no scene: no hold line (settings / enable logs already cover config).

### Invariants

Violate them and the feature is wrong.

**INVARIANT - hold latch (`bAlertHoldLogged` or equivalent):**

1. Set **`true` when a hold line is emitted.**  
2. Set **`true` on successful fire** - the fire line explains the episode; do **not** clear on fire (clearing causes the next tick to spam `alert hold why=already`).  
3. Reset to **`false` only on scene adopt** (same place `bAlerted` clears).  
4. Sleep-grace: separate latch or every-defer policy is an implementation pick - document it; do not flood Events every poll for a long grace wait without a latch.

**INVARIANT - elapsed expand:**

Wallclock elapsed is heal-capable (reload rebaseline + "last self-repair"). **v1 computes it up front every policy evaluation**, including post-alert ticks. Hold and fire **reuse those locals** - no second heal-capable read for decline classification.

- **Do not** re-introduce today's `!bAlerted` short-circuit to "preserve purity."  
- **Do** ship a **CHANGELOG** line: same reload rebaseline heal as before, now also on normal post-alert polls (not diagnostics-only).  
- Game-time stamp: still no wallclock-style rebaseline; only the rare negative-elapsed restamp if it ever appears.

### Poll tick (secondary)

Add `playing=` to the Every-check heartbeat, so the load-bearing `IsPlaying()` gate shows on the line already turned on for timing bugs:

```text
poll tick scene=0x… el=… thr=…s alerted=0|1 playing=0|1|- alive=1
```

`-` with no current scene, else `BoolField(currentScene.IsPlaying())`. One native read per tick, only when `iLogLevel >= LOG_CHECK` (Off/Events build nothing); no new quest field, no `RunCheck` change. Order stays log-**before**-`RunCheck` so the heartbeat is a timer-at-arm snapshot, unless detection has a strong reason to redefine it. Optional dual-clock fields on Every check only if the line stays one greppable row. **Not** a substitute for Events hold - Every check is not the bug-report path.

### Grep cheatsheet (target after 0.6.0)

```text
[fth_IJW] alert fire          # toast; see sig=
[fth_IJW] alert hold          # declined; see why=   (bug-report path)
deferred=sleep-grace          # primary deferred this check
[fth_IJW] poll tick           # Every check only; playing= / alerted= / el=
```

## Stance

False positives are patchable; silent save-killers are not. Prefer a dismissable wrong ping over a quiet broken save. Today's 3 min wallclock-only is too aggressive mainly because of **tight wall default + menu-pause**, not because "stuck" is wrong. Primary game-time + looser real fallback + short sleep grace is the detection fix. Re-alert is **opt-in noise**, not the default product personality - the menu is the reliable surface; the toast is a nudge.

**Bake in public:** defaults (2h game / 15m real / 3m grace / re-alert off / 5m cadence when on) are **starting numbers for the RC**, not sacred. Patch freely from community feedback on the road to 1.0.0. Do not delay 0.6.0 waiting for a perfect LO matrix.
