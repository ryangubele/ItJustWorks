# Ideas

Where the road ahead gets thought through out loud - our running conversation with ourselves, and eventually with the community, about where It Just Works goes next.

These are cairns on the trail, not milestones - mile markers dropped whenever the thinking is worth keeping. The folder is always just a snapshot: an idea that doesn't pan out gets deleted, and a versioned idea graduates out of here the moment it ships as code.

## The naming convention

- **`some_idea.md`** - named for itself: an idea worth keeping, not tied to a release.
- **`0.3.0_some_idea.md`** - carries a version, meaning we think it lands around there. That number is a **target, not a commitment** - "this is where we think it goes," not "this is promised." Order can shift, and anything here can still be dropped.

## What earns a version target

An idea graduates from named-for-itself to a dated target only when it's **grounded in a design justification that meets the design goals** in [`DESIGN.md`](../DESIGN.md). Not "would be cool," not "someone asked": a stated reason tied to a principle. A well-reasoned *exception* to a goal counts too, as long as the reasoning is on the page - the goals are a discipline, not a cage, and knowingly bending one with an argument is fair. If an idea can't yet say which principle it serves (or which it bends, and why), it stays named for itself until it can. Every versioned doc opens with a **Grounded in:** line, so the justification travels with the idea.
