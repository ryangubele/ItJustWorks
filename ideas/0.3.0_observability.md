# 0.3.0 - Observability: make it show its work

**Grounded in:** *Make it observable* (the practice), standing on five principles at once - respect the player, respect the contributor, never break a save, modern habits, and no ego (a tool that can't watch itself is claiming it never errs). Also the hard prerequisite for 0.5.0: a smarter detector is only as trustworthy as your ability to watch it decide.

The tool makes silent corrections. Every self-heal is mute today unless it happens to route through a `Trace` we already had - most of all the S-0 re-baseline (`ElapsedInScene` re-baselines a scene's stuck-timer after a full game restart so it still alerts instead of going silent forever). That is the save-safety fix that matters most, on the path hardest to reproduce, and it writes nothing. A fix you can't watch fire is a fix you can't verify. Scene bugs also only reproduce on real, heavy load orders; if the tool won't say what it saw or did, one careful reproduction becomes sixteen frustrated ones. This release makes the tool audible.

## The log format (committed)

Freeze this shape now, before 0.5.0 writes detector decisions into it:

```
[fth_IJW] <tag> <event> key=value key=value ...   (optional human note in parens, at end only)
```

- **`[fth_IJW]`** - the existing stable prefix, unchanged. Emitted once, inside the log primitive, never by callers. Every filter keys on it.
- **`<tag>`** - one token from a *closed* set: `life` `scene` `poll` `heal` `alert` `hotkey`. Reserved for 0.5.0: `det`. Adding a tag is a design decision made here, never a coding convenience.
- **`<event>`** - one stable lowercase token (`armed`, `enter`, `leave`, `rebaseline`, `fire`, `stop-req`, `stop-result`, ...).
- **fields** - space-separated `key=value`. **Every value is space-free and paren-free** - this is a hard contract, because a single space inside a value silently desyncs every downstream field for a `grep`/`awk` reader. Never log `GetQuestLabel`'s `edid (Display Name)` form; log a quest or scene as a bare editor id or `0x<formid>`. Empty renders as `-`. Values only - the one optional free-text `(...)` note is allowed at the end of a line and is never parsed.
- **Values are precise, never the readout's lossy phrasing.** The log carries `el=184s`, not `~3m` (`ElapsedLabel` rounds to whole minutes above 90s - useless for confirming a fix fired at 184 vs 180, or for tuning a threshold). Human rounding belongs in the readout only.

**Field glossary (stable names):**

| key | meaning |
|---|---|
| `scene=0x<formid>` | the join key. Native `GetFormID` via the existing `HexOf` (sign-safe, ESL-range-safe); present on every scene line, cheap, and the **same id the MCM Form ID row shows** - so a player can grep the log for the exact scene in front of them. `-` when no scene. |
| `name=<edid>` | human editor id, via a side-effect-free fetch. Only on the *rare* Normal lines (enter/leave/alert/stop/heal); `-` when Load EditorIDs is off. **Never on the per-poll line.** |
| `el=<int>s` | elapsed seconds in scene. `thr=<int>s` active alert threshold. |
| `was= now= dt=` | heal rebaseline: stamp before / after / the jump erased (seconds). |
| `len_was= len_now=` | heal history-ring length before / after. |
| `via=` | player-ref recovery route. `cleared=0\|1` stop outcome. |
| `alerted=0\|1 alive=1` | verbose heartbeat state. |
| `player= hist= hotkey= level=` | `life armed` startup status. `poll= warn= level=` `life settings`. |

Reserved for 0.5.0 (`det` subsystem, names fixed now, emitted later, **zero grammar change**): `gt=<float>d` game-time in scene (second clock), `tgt=<float>d` its threshold, `long=0|1` long-scene-list member, `sig=wall|game|both` which clock tripped, `trip=`.

**Examples (0.3.0):**

```
[fth_IJW] life armed player=ok hist=new hotkey=off level=0
[fth_IJW] scene enter scene=0x000ABCDE name=DialogueFollowerExtraScene el=0s
[fth_IJW] heal rebaseline scene=0x0D0F1A2B was=13571 now=2 dt=13569 (real-time reset across reload)
[fth_IJW] alert fire scene=0x0D0F1A2B name=MG07BootScene el=184s thr=180s sig=wall
[fth_IJW] alert stop-result scene=0x0D0F1A2B cleared=1
[fth_IJW] poll tick scene=0x0D0F1A2B el=120s thr=180s alerted=0 alive=1       (verbose only)
```

The same alert line at 0.5.0, no parser change:

```
[fth_IJW] alert fire scene=0x0D0F1A2B name=SeasonUnending el=1210s gt=5.2d thr=1200s tgt=5.0d sig=game long=1
```

`grep heal` = every self-heal that fired. `grep 0x0D0F1A2B` = one scene's whole life (enter -> tick -> fire -> stop) across a reproduction. Subsystem filtering lives in the *line*, greppable - never as MCM toggles.

**Transport: `Debug.Trace` into the shared Papyrus log.** Not a dedicated user log. A user log needs `Debug.OpenUserLog` once per session, but this is a `Quest` script with no `OnPlayerLoadGame` - it could only reopen at `OnInit` and on MCM-open, and the S-0 `heal rebaseline` fires across a reload *before* any MCM-open. A dedicated sink would drop exactly the line the whole release exists to make visible. `Debug.Trace` captures it unconditionally, and `findstr fth_IJW` gives the isolation a dedicated file would.

## Verbosity: one setting, three levels

A single enum, **not** per-subsystem flags - a toggle matrix is the general-logging-framework this mod must never become. One `iLogLevel` int (`ModSettingInt`) replaces the `bTrace` bool:

- **Off (0)** - nothing reaches the log. The shipped default (matches today's `bTrace=false`; logging is opt-in for a bug report, never a tax on normal play).
- **Events (1)** - every state transition, every correction/self-heal, every alert and user action. Event-bounded: fires per scene-change / heal / action, never per tick.
- **Every check (2)** - Events plus the per-poll `poll tick` heartbeat (loop-alive proof, threshold-approach watch). The only per-cadence line.

The **rule that keeps future code (and 0.5.0) consistent:** a line is *Events* if its cause happens at most once per scene-change, heal, or user action; it is *Every check* if it happens on the poll cadence. Emit a line only when the tool (a) changed its own state, (b) reconciled a state it found wrong, or (c) took an action the player can perceive. A pure read that changes nothing gets no line at Events. This is the anti-balloon clamp.

MCM: replace the `bTraceLog` toggle on the Diagnostics page with one option-list control, labels **Off / Events / Every check**, help text that says what each writes and points at the "turn the log on" manual section (the setting is dry without the ini flags).

## What gets logged, and when

| tag | event | level |
|---|---|---|
| life | armed (OnInit) - `player=ok\|recovered hist= hotkey= level=` | Events |
| life | enabled / disabled (master switch) | Events |
| life | settings applied (**guard on a real change** - a slider drag re-fires per step) | Events |
| scene | enter / leave (leave carries `el=` duration) | Events |
| heal | **rebaseline** - S-0 re-baseline after real-time reset (`was/now/dt`) — the marquee | Events |
| heal | **hist** - EnsureHist rebuilt a wrong-length ring (`len_was/len_now`) | Events |
| heal | **player** - PlayerRef recovered when the VMAD fill was empty (`via=`) | Events |
| heal | **reassert** - a *detected* dropped loop was re-armed (see below) | Events |
| alert | fire (`el thr sig`) | Events |
| alert | stop-req / stop-result (`cleared=`) | Events |
| hotkey | name - bound key named the current scene | Events |
| poll | tick - heartbeat (`scene el thr alerted alive=1`) | Every check |
| heal | reassert (routine) - blind re-register, no drop detected | Every check |
| life | hist-init - first lazy ring build (benign one-time) | Every check |

## The performance fix (it closes a real bug today)

Papyrus evaluates a call's arguments **eagerly**: `Trace("enter " + LabelFor(x))` builds the whole string *and calls `LabelFor`* even when logging is off - and `LabelFor` has a side effect (it fires the one-time "names are off" notification and sets `bEditorIdHinted`). So today's off-path can trigger player-facing UI from a logging call. The guard fixes this structurally:

- **Gate the string *build*, not just the write, at the call site.** Wrap any concatenating or helper-calling line in `if iLogLevel >= LEVEL` before it builds. Below that level the line costs exactly one int compare - zero allocation, zero SKSE call, zero side effect. The primitive re-checks for bare-literal callers: `Function Log(int aiLevel, string asLine) if iLogLevel >= aiLevel Debug.Trace("[fth_IJW] " + asLine) endif`.
- **Two side-effect-free helpers keep the hot path clean.** `SceneKey(scene)` -> `0x`+`HexOf(GetFormID())` or `-` (native, no edid, no hint) - used on the per-poll heartbeat and as the `scene=` field everywhere. `QuietEdid(scene)` -> `GetFormEditorID` with **no** hint notification - used only on rare Events lines, never per-poll.
- **The verbose heartbeat is formid-only.** It never calls the SKSE edid path; over a long "Every check" session it's one short line per poll (~120/hour at the default 30s), opt-in, no rotation or throttle needed - the level gate is the guard.

Net: Off is now *cheaper and side-effect-free* versus today; Events is event-bounded so it can't grow with session length; Every check's only per-cadence cost is one `SceneKey`+concat per poll, paid solely when chosen.

## Watching the loop stay alive

The one thing the log can't answer for a player who won't open it: *is the watchdog even running right now?* One persisted float, `fLastTickRealTime = Utility.GetCurrentRealTime()`, stamped in **`OnUpdate` only** - the real timer path, never `RunCheck` (which also runs out-of-band on MCM-open and inside `StopCurrentScene`; stamping there would refresh the clock every time a worried player opens the menu, so a dead loop would read as alive - the indicator would mask the exact failure it exists to reveal). The Diagnostics readout renders it in calm words: **Running (~Ns ago) / Off / Dormant / Waking up / Late** - never a caps "STALLED" on what is usually a routine post-reload lag. Guard the subtraction for the real-time reset exactly as `ElapsedInScene` does (now < stamp -> "Waking up", not a negative or huge number).

The same stamp earns a second thing honestly. `ReassertRegistrations` re-registers key+timer on every MCM-open and can't natively tell whether anything was dropped - so a blanket "I healed something" line there would claim a fix it can't prove (a No-Ego violation). Instead: if the loop is enabled and `now - fLastTickRealTime` exceeds ~2x the poll interval, the loop *was* dead -> emit `heal reassert` at Events (a real, detected correction). Otherwise it's routine -> `Every check` only. **Exclude the post-reload waking window** (the same now < stamp guard), or the honesty mechanism would fabricate a phantom recovery every time you open the menu shortly after a load.

## The MCM readout

The division of labor: the **log** is the debugger's lossless, ordered, greppable transcript (history, exact values, per-poll ticks); the **readout** is the player's present-tense *now* (live state + is-it-working), never a stream of the log. They meet at exactly one point - both print the identical `0x<formid>` scene id.

0.3.0 adds, on the Diagnostics page, exactly three things:

- the **Log level** control (Off / Events / Every check), replacing the Trace toggle;
- a **loop-alive** row (Running / Off / Dormant / Waking up / Late), from `fLastTickRealTime`;
- a **Last correction** row - a calm phrase naming the most recent *real* self-heal ("Re-synced the scene timer after a reload", "Reconnected to the player", "Rebuilt the recent-scenes list"), or `-` when none has fired. This is the one place the practice is non-negotiable: *every correction the tool makes is visible somewhere you can reach without a debugger, on the readout while you play*. The S-0 re-baseline that guards the prime directive fires silently today; here it finally says so - to the player who will never open a log.

**The boundary, held deliberately - one row, not a panel.** The row shows *what* it last did, not *when*: label only, because the log carries the precise timing for anyone digging, and dropping the age also drops a persisted timestamp, an extra reload-guard, and a translation-fragile localized "~2m ago". No checks-this-session counter (that's telemetry, not repair), no always-visible status badge on the Scene page (a mod named *It Just Works* doesn't narrate itself to players who never asked), no separate last-alert row (the alert already showed on screen when it fired). Correction *history* and exact values stay in the log; the readout carries live state, is-it-alive, and the single last thing it fixed.

**The alarm rule (load-bearing):** Last correction updates ONLY for heals that actually changed something - the S-0 re-baseline, PlayerRef recovery, a history rebuild - never the idempotent re-arm that runs on every menu-open, or the mod would read as constantly breaking. The help text frames a correction as normal, healthy housekeeping, not a fault.

**The cost, named plainly:** one net-new persisted field beyond the loop-alive float - an inert label (`$`-key) string, no world handle, self-healing default, the same risk class as the already-persisted `fSceneFirstSeen` and history ring. It bends *hold nothing* by exactly that one string. That is the minimal price the *Make it observable* practice charges - and that practice is the reason this whole release exists.

## Documenting the pipe

The log writes nothing until Papyrus logging is enabled in the game ini - a tool that claims observability has to hand you the tap.

- **Manual - "See what it's doing":** add a `[Papyrus]` block to `Skyrim.ini` (or `SkyrimCustom.ini`): `bEnableLogging=1` and `bEnableTrace=1` (only these two - `bLoadDebugInformation=1` only enriches stack traces and adds overhead). The file lands at `Documents\My Games\Skyrim Special Edition\Logs\Script\Papyrus.0.log`; filter it with `findstr fth_IJW` (or `grep`). MO2 note: it's the real Documents folder, not the virtualized game dir. Set the MCM Log level to Events (or Every check for a timing bug), reproduce once, read the log.
- **Contributor note (DESIGN.md's *Make it observable* practice):** the line-format contract - grammar, closed tag set, field glossary, the emit-a-line rule, the Events-vs-Every-check rule, the space-free-value hard rule, and the guard idiom - stated as a contract 0.5.0 must not break.

## Forward-serving 0.5.0

The detector is a judgment machine (two clocks, a relaxed backstop, per-scene thresholds, a long-scene list); tuning it means answering "which signal tripped, at what value, on which scene, and was that right" over and over. The committed format already carries the answer: `det` slots in as a new tag, the reserved `gt=/tgt=/long=/sig=/trip=` fields append to `alert`/`poll` lines with no grammar change, per-scene thresholds simply change the `thr`/`tgt` values, and precise integer/float values (never `~3m`) make it tunable. Commit the format now; fill the fields at 0.5.0.
